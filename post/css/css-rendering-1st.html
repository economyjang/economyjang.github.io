📍Graphics System

Graphics System 의 정의
  • Graphics System이란 화면에 점을 찍는 방법을 말한다.
Graphics System 이 점을 표현하기 위해 필요한 기본 요소
  • X, Y, width, height, color 등이 있다.
Graphics System 이 점을 얼만큼 표현할지 계산하는 방법
  1. Fixed Number
    • 고정된 값을 이용해서 점을 표현한다.
    • 고정된 사이즈를 위한 전용 시스템을 만들 때는 문제가 되지 않았으나, 여러가지 환경에서 사용하기 위한 시스템은 해상도가 전부 다르기 때문에 문제가된다.
  2. Abstract Calculator
    • 스크린 사이즈에 따라 다르게 값을 계산해서 점을 표현한다.
    • 고정된 값을 주지 않고, %, left, right, block, inline, float 등의 화면을 계산하는 계산식 또는 공식을 사용한다.
  3. Components
    • 추상화된 Graphics System 을 이어받아 공통적인 특성을 갖고 있는 것을 말한다.
    • ex) 각각의 html tag, bootstrap component
  4. Framework
    • 만들어진 Components 가 일정한 규칙을 갖고 있는 것을 말한다.
    • ex) html 전체 tag, bootstrap

📍Normal Flow

  • Visual formatting model - Normal Flow 참고 문서
Normal Flow 의 정의
  • 어떻게 화면에 Visual Formatting 을 할 것 인지에 대한 방법
Normal Flow 에 해당하는 Position 의 요소
  • Static, Relative
Normal Flow 를 계산 공식 - Normal Flow 는 밑의 3가지 속성을 기준으로 그려진다.
  1. Block Formatting Contexts
    • 하나의 Block 은 부모의 가로 길이를 가득채운 한 줄 이다.
    • Block 내부의 여러개의 Block 이 존재할 때 내부 Block 의 높이를 합친 값이 외부의 Block 높이가 된다.
  2. Inline Formatting Contexts
    • 나의 컨텐츠 크기만큼 가로를 차지한다.
    • 다음 Inline 의 위치는 바로 앞 Inline 요소가 끝나는 지점부터 시작한다.
    • Inline 요소들의 길이 합이 부모의 길이를 넘어가면 밑으로 내려가서 Inline 요소가 그려진다.
    • Inline 요소 다음 Block 요소가 시작되면 밑에서 새롭게 시작한다.
    • 공백문자가 없는 한줄의 문자열은 하나의 Inline 으로 인식한다.
    • line height - 알아볼 것
  3. Relative Postion
    • Static Postion 으로 그려진 다음 상대적으로 위치를 움직인다.
    • Relative Postion 은 Static Postion 위에서 움직이기 때문에 겹치는 현상이 생긴다.
    • html 모든 요소는 기본 Static Position 이다.
    • 이외에 Fixed, Absolute, Sticky Position 을 선택하면 Normal Flow 를 벗어나기 때문에 기존의 속성들이 사라진다.

Normal Flow 대로 (Static) 전부 그리고 상대적으로 Relative Position 만큼 위치를 이동시킨다. Static 에서 Relative Position 이 작동하면 Static 위를 덮게 된다. (Relative Position 끼리는 z-index 로 경합)

 

📍Float

Float 의 정의
  • Float 는 BFC 를 띄워서(floating) 어디에 위치시킬지 결정하는 요소이다.
Float 의 속성
  • left, right, none, inherit 속성이 존재한다.
Float 의 특징
  1. New BFC - 새로운 BFC 를 만든다.
  2. Floating over Normal Flow - Normal Flow 위에 떠서 위치를 결정한다.
  3. Text, Inline Guard - Text, Inline 요소의 guard 로서 작동한다. (Text, Inline 요소는 밀어내지만 Block 은 밀어내지 않는다.)
    • content 가 커서 BFC Box 속성이 overflow: visible 일 때 Box 크기가 커지지만, Guard 로 인해서 content 가 밀려나면 BFC Box 가 늘어나지 않는다.
  4. Line Box - Normal Flow 와는 달리 Line Box 기준으로 위치를 결정한다.

 

New BFC + Float

<div style="width: 500px;"> <!-- 첫 번째 BFC 영역시작 -->
  <div style="height: 50px; background: red"></div> 
  <!-- 첫 번째 BFC 영역에 그려짐 -->
  <div style="width: 200px; height: 150px; float: left; background: rgba(0, 255, 0, 0.5)"></div> 
  <!-- float 속성으로 인해 새로운 BFC 를 생성하고, 그 안에 그려짐 -->
  <div style="height: 50px; background: skyblue"></div> 
  <!-- 새로 생성된 BFC 영역 안에서 그려짐 -->
</div>

첫 번째 BFC 영역이 등장하고, 이후 첫 번째 Red Box 는 일반적은 Normal Flow 라서 내부에 그려진다. 두 번째 Box 는 Float 속성을 갖고 있기 때문에 첫번 째 BFC 영역 다음에 새로운 BFC 영역을 만들고, 내부에 Box 를 그린다. 세 번째 Box 는 두번 째 BFC 영역에 그려지지만 Float Box 는 Normal Flow 속성을 가진 Box 위에 둥둥 떠있는 속성을 갖고 있기 때문에 겹쳐서 그려진다.

 

Text, Inline Guard

<div style="width: 500px;">
  <div style="height: 50px; background: red"></div>
  <div style="width: 200px; height: 150px; float: left; background: rgba(0, 255, 0, 0.5)"></div>
  Hello
  <div style="height: 50px; background: skyblue">World</div>
  !!
</div>

Hello, World, !! 는 왼쪽에 붙어서 그려져야 하는데, Float Box 가 Text, Inline 요소의 Guard 로 작동해서 밀어내고 있기 때문에 Float Box 옆에 그려진다. 단 Text, Inline 요소의 Guard 로 작동하지만, Box 요소의 Guard 로 작동하지 않는다.

 

Line Box - Line Box 의 제일 left or right 에 붙는 것이 Float 의 속성

  <div style="float: right; width: 50px; height: 150px; background: red;">2</div>

  <div style="height: 30px; background: red;">ABC1 ABC2 ABC3 ABC4 ABC5 ABC6 ABC7 ABC8</div>

가용한 Line Box 안에서 최대한 left 또는 right 에 Box 를 위치시킨다. 하지만 이미 그려진 Float Box 있다면 그보다 더 left, right 에 그려질 수 없다. 만약 Line Box 보다 큰 width 가진 Float Box 를 그리려고 할 때는 Line Box 의 크기가 넘치므로, 현재 Line Box 의 바닥면을 기준으로 한 빈 공간이 새로운 Line Box 의 영역으로 설정하고 Float Box 를 위치시킨다. Float Box 를 위치시키고 남은 공간은 실제로 사용할 수 없는 죽은 공간이다. 제일 마지막 Normal Flow Box 가 Text 양쪽 공간이 짤려보이지만 사실은 width 전체를 차지하고 있다.

ABC8 은 바로 위 Line Box 는 Float Box 가 자리를 전부 차지하고 있기 때문에 바로 밑에서 그려진다.

 

📍OverFlow

OverFlow 의 정의
  • Geometry 를 넘어가는 요소를 어떻게 그릴지 결정하는 요소
OverFlow 의 속성

  • Visible, Hidden, Scroll, Inherit, Auto

  • 기본 값은 대부분 Auto(= Visible) 이다. Auto 는 내부의 요소가 커지면 부모의 크기도 커지는 속성이다. 즉 Geometry 를 키운다.

  • Scroll 은 자신의 Geometry 를 넘어가면 Scroll bar 를 만드는 속성이다.

  • Hidden 은 자신의 Geometry 를 넘어가면 안보이게 짤라버리는 속성이다.

  • Hidden, Scroll 속성만 Float Box 와 관계성이 있다.
